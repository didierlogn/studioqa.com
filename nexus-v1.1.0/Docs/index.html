Welcome to Our User Guide

Overview

Welcome to our user guide. Whether you are a developer or a web master, our guide is written to be as user-friendly and un-opinionated as possible, ensuring that all of our customers can learn how to make the most from our products. Although we go through great lenghs to ensure our products are as simple to use as possible, sometimes you may need to make a quick reference to this documentation to check how something is done.

This product is a combination of HTML, CSS/SCSS and JavaScript files. This user guide will show you everything you need to know about each of these files and how they interact with other other to create your amazing new website. Although you don't need to be an expert developer to customize our HTML templates, you will need at least a basic understanding of how HTML works if you plan on creating a custom website with this product. Of course, if you are a developer, we also go into the advanced stuff as well.

Optimisation/PerformanceAdvanced

Whilst this theme is ready to go out-the-box, we recommend taking a few steps to improve the overall performance and optimisation of this theme before using it in a live environment. There are many features included that are not used by default (but are there in case you plan on building upon this theme) and all included customizable assets are provided unminified and unconcatenated.

  1. Compress/minify assets/styles/css/app.css
  2. Combine/concatenate all JavaScript/jQuery files into one minified file
  3. Remove any undesired SCSS partials and recompile app.css
  4. Consider creating custom concatenated js files for each page

Homepage Options

Nexus comes with four default homepage options, each one giving a different unique first-impression to your visitors. Choose between various parallax/scrolling options and an HTML5 video option. All of the below options are controlled by the homepage.js file.

  • assets
  • js
  • homepage.js

Parallax Title (Default)

This is the default option for the homepage, and is achieved by adding the class parallax-title to the body element of your homepage. This adds a parallax effect to the title-wrapper element.


<body class="index parallax-title">

Parallax Background

To add a parallax effect to the background image instead, add the class parallax-bg to the body element.


<body class="index parallax-bg">

Fixed Background

To created a fixed background and title, add the class hero-fixed to the body element. This causes the entire hero section to remain fixed, allowing the content to scroll over it.


<body class="index hero-fixed">

Video Background

To use your own video as the background for the hero section, first add the class video-bg to the body element.


<body class="index video-bg">

Next, you must ensure you have the HTML5 code containing your video located directly after the closing header tag of the main app-header.


...
</header>

    <video autoplay loop poster="../../assets/images/YOUR-IMAGE-BG.jpg" class="hero-vid">
        <source src="../../assets/videos/hero-vid.mp4" type="video/mp4">
    </video>
    <div class="hero-overlay"></div>

Ensure that the paths to the background image fallback and video itself correspond to the files you are using.

Color Scheme

By default we have provided four different color schemes for you to choose from, but with the simple change of just one Sass variable you can change the entire theme to be whatever color you want.

Default Colors

To use one of the default colors, delete or rename the default app.css and then rename the file for your desired color to app.css. If you are using Sass, copy the contents of app-yourcolor.scss into app.scss, ensuring you overwrite everything that was there pveviously. If you want to use the default color, blue, there are no changes you have to make, but you can delete the redundant CSS/Scss files for the other colors if you wish.

Custom Color Advanced

To use your own color, open the variables file and locate the brand-1 variable. Change this value to any color you want; you can use hexadecimal, rgba, hsl and any other valid CSS format. This single value alone controls the entire main color scheme for the theme.

  • styles

Off-Canvas Navigation

The HTML for both off-canvas menus is created from the main menu, so you only have to maintain one copy.

Control Panel Naviation

The Control Panel Navigation is a modern and unique control panel used for desktops and devices with large resolutions. The control panel contains the website's main menu, search bar, logo and copyright information. All of these items are dynamically created from the existing content, so there's no need to maintain two seperate copies. This panel is only avilable when the fixed header option is disabled. The control panel nav has its own Scss and JS files:

  • assets
  • styles
  • scss
  • modules
  • _control-panel-nav.scss
  • assets
  • js
  • control-panel-nav.js

To toggle the control panel nav, just add the cp-trigger ID to any element. By defult, there is control panel trigger button which appears on page scroll. Clicking this button toggles the cp-active class on the body element, triggering the animations revealing the panel.


<span class="nav-trigger cp-trigger" id="cp-trigger"></span>

Panel Construction Advanced

Each element in the control panel is a clone of the original element from the existing HTML markup, removing the need to maintain two versions of the same element. They are inserted into a pre-existing element with an ID of cp-nav.

Example:


var cpContainer = $('#cp-nav');
...
$("#app-header .main-nav > ul").clone().appendTo(cpContainer);

Flyout Navigation

The Flyout Navigation is the system used for mobile devices (by default) and browsers which don't support the Control Panel Navigation. The menu supports infinite nesting of child menu items. The flyout nav can also be used instead of the control panel nav if you prefer a more conventional approach.The flyout nav has its own Scss and JS files:

To enable the fixed header, add the class fixed-header to the body element of your page.

Google Maps

To use the custom Google Maps as shown in the demo, you will need your own API key. Please visit the Google Maps API page to get your key. Once you've got your key, locate the following code in a page which has the Google Map:


<!-- Google Map -->
<script type="text/javascript" src="https://maps.googleapis.com/maps/api/js?key=YOURKEY&sensor=false"></script>

Replace YOURKEY with your unique API key. To customize the default map options, edit the google-maps.js file:

  • assets
  • js
  • google-maps.js

Previews Carousel

The single-project.html page displays any number of preview images for your project in our custom designed carousel, functioned by the Owl Carousel plugin. 


/******************************************************************
Portfolio Previews Carousel
******************************************************************/

$('.single-project .previews').owlCarousel({
    items: 1,
    dots: false,
    loop: true,
    nav: false,
    autoplay:true,
    navText: [],
    slideBy: 1
})

var pp = $('.single-project .previews');

pp.owlCarousel();

// Go to the next item
$('.pp-next').click(function() {
    pp.trigger('next.owl.carousel');
})

// Go to the previous item
$('.pp-prev').click(function() {
    pp.trigger('prev.owl.carousel');
})
